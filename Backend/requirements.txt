# WHY THESE VERSIONS:
# - numpy/scipy/pandas/sklearn/xgboost must all be binary-compatible.
#   Loose or unpinned versions cause "build fail" or silent crash on Render
#   because pip resolves a combo that doesn't match your trained .pkl files.
# - These versions are tested together and work on Render's Python 3.11 image.
# - If your model was trained with a DIFFERENT scikit-learn/xgboost version,
#   you MUST retrain and re-save the .pkl files with THESE versions,
#   otherwise joblib.load() will throw "incompatible version" errors silently
#   (your code catches it, but model becomes None → fallback mode forever).
# ─────────────────────────────────────────────────────────────────────────────
